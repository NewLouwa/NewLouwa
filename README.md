# A sysadmin who peaked during the Minecraft server era

Once upon a time I wrote HTML on actual paper during history class. My teacher was talking about World War I. I was debugging a `<table>` layout. We were not the same.

---

## 🕹️ How we got here

It started with a Minecraft server at 12. I thought I was playing a game. I was actually getting a free crash course in Linux, networking, and conflict resolution with angry teenagers. If it had a config file, I edited it. If it didn't, I found one anyway.

Web dev came right after and stuck around for six years. HTML, CSS, JS, PHP, frameworks on both sides of the stack. At 13 I proudly opened `localhost:8080` and sent the link to a friend. He couldn't open it. I spent the next two days figuring out why. We don't talk about those two days, but I think about them every time someone asks me what a reverse proxy does.

Meanwhile, in the physical world, ⚡ Arduino happened. Things blinked. Things beeped. Things got plugged into wall sockets that probably shouldn't have been. I built a DIY Bluetooth speaker from an amp and a sketchy module and it worked, technically, for a while. Some things caught fire. Most didn't. I'm counting that as a win.

Software dev followed naturally. C, C++, C# in Unreal, then Python took over, and now Rust and Go are elbowing their way in.

Somewhere along the way I realized I cared less about shipping the thing and more about figuring out how it worked underneath. Why it worked. What happened if you poked it wrong. Then I watched Mr. Robot and thought "wait, that's just me". 🔓 The show didn't make me want to hack, it just put a name on something I was already doing.

Hacking rabbit hole. Open source rabbit hole. And then a decision that sounds backwards but isn't: I went into sysadmin on purpose. You need the keys before you learn to pick the lock. Best entry point into cybersecurity I could've picked, and I wouldn't redo it any other way.

---

## 💻 What I'm actually doing

🏢 **Day job:** Sysadmin. Keeping servers alive, writing AD scripts, and standing between users and their own worst instincts. Meat is always the weakest point in the stack and I try not to be A5 wagyu.

🌙 **Night job:** HackTheBox, homelab, automating everything that stands still long enough. Breaking into the kind of things I spend my days protecting.

🍪 **Personality trait:** If I have to do something more than twice, it becomes a script. I've automated AD user management, system configs, and Cookie Clicker stock trading. I will not be taking questions on that last one.

🔄 **Other personality trait:** Every time I need a tool, I build it from scratch. Then I find out someone already shipped it, documented it, and is hosting it for free. Every. Time.

🎨 **Side quest:** ComfyUI custom nodes, because every model has training parameters and hunting them down per workflow is a war crime. I really need to get back to that one.

- 📓 HTML in notebooks during history class

- 🌐 Web dev (age 13, learned localhost:8080 doesn't mean the internet can see it)
- ⚡ Arduino & electronics (meanwhile, some things caught fire)
